DATABASE SCHEMA — implement exactly
═══════════════════════════════════════════════════════

Table: users
  user_id    SERIAL PRIMARY KEY
  email      VARCHAR(255) UNIQUE NOT NULL
  password   VARCHAR(255) NOT NULL          -- bcrypt hashed
  f_name     VARCHAR(100) NOT NULL
  l_name     VARCHAR(100) NOT NULL
  company    VARCHAR(255)
  created_at TIMESTAMPTZ DEFAULT NOW()

Table: sessions
  session_id SERIAL PRIMARY KEY
  user_id    INTEGER REFERENCES users(user_id) ON DELETE CASCADE
  title      VARCHAR(255) DEFAULT 'New Chat'
  created_at TIMESTAMPTZ DEFAULT NOW()

Table: chat_history
  id         SERIAL PRIMARY KEY
  session_id INTEGER REFERENCES sessions(session_id) ON DELETE CASCADE
  role       VARCHAR(10) NOT NULL CHECK (role IN ('user','agent'))
  content    TEXT NOT NULL
  media_url  TEXT                             -- for images/files from n8n
  created_at TIMESTAMPTZ DEFAULT NOW()

History fetch:
  SELECT * FROM chat_history WHERE session_id = ? ORDER BY created_at ASC

═══════════════════════════════════════════════════════
AGENT ROUTING — CORE LOGIC
═══════════════════════════════════════════════════════
The chat input has an agent selector. Options:
  @CRM        → POST to N8N_CRM_WEBHOOK_URL
  @NewsLetter → POST to N8N_NEWSLETTER_WEBHOOK_URL
  @Xero       → POST to N8N_XERO_WEBHOOK_URL
  General     → Use Claude / OpenAI API directly (no n8n)

When an agent is selected, ONLY that agent is active for the session until changed.
The selector is prominently displayed and clearly shows which agent is active.

Payload sent TO n8n (JSON):
{
  "user_id"    : <int>,
  "session_id" : <int>,
  "message"    : "<string>",
  "history"    : [{"role": "user"|"agent", "content": "..."}],
  "file_url"   : "<string|null>"       // if file/image attached
}

Response FROM n8n (JSON):
{
  "user_id"    : <int>,
  "session_id" : <int>,
  "message"    : "<string>",           // original user message
  "reply"      : "<string>",           // agent reply to display
  "media_url"  : "<string|null>"       // optional image/file URL to render
}

After every exchange, append 2 rows to chat_history:
  {role: 'user',  content: message,     session_id}
  {role: 'agent', content: reply,       session_id, media_url}

═══════════════════════════════════════════════════════
HISTORY MANAGEMENT
═══════════════════════════════════════════════════════
- n8n has NO memory. Feed it the FULL history every call.
- Summarisation: if history exceeds 30 turns, compress the oldest 20 turns
  into a single summary message via a second LLM call, replace those rows
  with one {role:'agent', content:'[Summary] ...'} row. 
  Implement in summarisation_service.py.
- History is fetched fresh from DB before EVERY message send.

═══════════════════════════════════════════════════════
SESSION TITLES
═══════════════════════════════════════════════════════
- On the FIRST user message of a new session, call the LLM to generate
  a concise title (≤6 words) from that first message.
- Update sessions.title with the result.
- Sidebar updates reactively without full reload.
